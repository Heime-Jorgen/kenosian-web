What It Is

AI Agent는 컨텍스트 압축(context compaction)이 발생할 때마다 이전 작업을 잊는다. Letta, LangGraph, Mem0 등 기존 메모리 솔루션은 vector embedding에 의존하기 때문에 fuzzy recall만 가능하고, 세션 재시작 후 결정적 상태 복구를 보장하지 못한다.

Hydra-MCP(OpenTTT)는 이 문제를 다른 레이어에서 해결한다. 모든 이벤트에 TTTPS temporal proof(PoT)를 봉인하고, causal chain으로 연결하여 SHA-3 해시 기반 O(1) exact lookup을 가능하게 한다. 서버 재시작 후에도 Redis DAG에서 전체 이벤트 체인을 재건한다 — 암호학적으로 검증 가능하게.

핵심 차별점

  • · OpenTTT MCP 서버 — Claude / GPT / Cursor에서 npx @helm-protocol/openttt-mcp 즉시 실행
  • · 8개 MCP 툴 — pot_generate · pot_query · pot_checkpoint · pot_verify · pot_status · pot_graph · pot_chain · pot_evict
  • · Long-Horizon Amnesia Zero — 1,000개 causal chain 검증, 서버 재시작 후 복구율 100%
  • · SHA-3 기반 O(1) exact recall — embedding cost 없음. vector DB 없음. 퍼지 검색 없음.
  • · IETF-track 표준 — TTTPS (draft-helmprotocol-tttps-03) 기반. 유일한 구현체.
  • · 오프라인 동작 가능 — Roughtime 서버 없이 local-clock 폴백 자동 전환
  • · signing_key 영속 — pub_key 변화 없이 세션 재시작 후에도 이벤트 복구 무영향

Fuzzy semantic recall이 필요하면 OpenTTT와 vector DB를 병행 사용. 결정적 상태 복구(deterministic state recovery)가 필요하면: OpenTTT가 유일한 IETF-track 구현체다.

Architecture

  • · Temporal Proof (PoT) — Roughtime multi-source (k≥3) 또는 local-clock 폴백으로 이벤트 타임스탬프 봉인
  • · SHA-3 causal chain — 각 이벤트의 해시가 다음 이벤트의 parent로 연결됨. O(1) exact lookup.
  • · Redis DAG 영속성 — AOF 모드로 서버 재시작 후 전체 체인 재건 가능
  • · Rollup 체크포인트 — 240 epochs 마다 자동 체크포인트. 오래된 이벤트 압축.
  • · MCP 인터페이스 — Model Context Protocol 표준. 어떤 AI 클라이언트에서도 동작.
  • · mode:local — 네트워크 없이 완전 오프라인 동작. Roughtime 서버 불필요.

설치 — 2분

서버 설치 불필요. npx 한 줄로 즉시 실행.

# npm / npx 직접 실행
npx @helm-protocol/openttt-mcp

# Claude Code에서
/mcp add openttt-mcp

# 또는 Claude Desktop settings.json
{
  "mcpServers": {
    "openttt-mcp": {
      "command": "npx",
      "args": ["@helm-protocol/openttt-mcp"]
    }
  }
}

설치 후 Claude/GPT에서 바로 사용:

# 이벤트 생성
pot_generate({ "content": "architecture decision: QUIC transport confirmed", "context": "session-001" })

# O(1) exact 조회
pot_query({ "event_id": "sha3-abc123..." })

# 세션 체크포인트 봉인
pot_checkpoint({ "label": "v0.3.0-release" })
