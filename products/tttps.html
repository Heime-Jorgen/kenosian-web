GRG Pipeline

Cryptographic commitment architecture — abstracted for IP protection.

Multi-source time anchors → cryptographic commitment pipeline → Ed25519 signature.
All manipulation converges to commitment mismatch.

The pipeline ingests independent time sources, computes a layered commitment across entropy, ordering, and integrity dimensions, and seals the result with an Ed25519 signature over the final chain digest. Any tampering at any stage produces a verifiable mismatch at verification time — regardless of which attack vector is used.

Step 1: Generate a token

curl -X POST https://api.kenosian.com/api/demo/pot/generate \
  -H "Content-Type: application/json" \
  -d '{"ctx_id":"demo"}'

Expected response:

{
  "status":         "ok",

Step 2: Verify (replay attack detection)

curl -X POST https://api.kenosian.com/api/demo/pot/verify \
  -H "Content-Type: application/json" \
  -d '{"ctx_id":"demo","token":"<paste token from step 1>"}'
