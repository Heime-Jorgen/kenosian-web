TTTPS Audit Integration

Every content chunk generates a TTTPS seal — cryptographically binding delivery to a verifiable timestamp and commitment chain.

Audit trail per chunk:

{
  "chunk_hash":    "sha256:a3f9c2...",
  "ctx_id":        "cdn-pool-sg1",
  "timestamp_ms":  1748000000000,
  "attack_type":   null,
  "signature":     "ed25519:..."
}

Queryable by audit organizations:

curl "https://api.kenosian.com/audit?ctx_id=cdn-pool-sg1&window=86400" \
  -H "X-API-Key: <your-key>"

One-Click Test — Delivery Integrity Check

# Check delivery integrity of a chunk
curl -X POST https://api.kenosian.com/api/demo/pot/generate \
  -H "Content-Type: application/json" \
  -d '{"ctx_id":"cdn-demo"}'

Expected: TTTPS seal generated, GPS/BGP/NTP attack detection active

Replay attack simulation:

# Submit same token twice to trigger REPLAY detection
curl -X POST https://api.kenosian.com/api/demo/pot/verify \
  -H "Content-Type: application/json" \
  -d '{"ctx_id":"cdn-demo","token":"<same-token>"}'
# → {"valid":false,"attack_type":"REPLAY"}
